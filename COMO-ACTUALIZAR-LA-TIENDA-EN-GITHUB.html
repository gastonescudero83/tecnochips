Cómo actualizar tu tienda en GitHub

Guía paso a paso para subir tus cambios vos mismo, sin saber programar.

Cada vez que cambies un archivo de la tienda en tu computadora (por ejemplo un .js, un .css o el index.html), tenés que subir ese archivo a GitHub para que se vea online. Se hace desde la página web de GitHub, arrastrando archivos. No hace falta instalar nada.

Antes de empezar necesitás 1) Tu carpeta tienda-online en la PC.   2) Estar con tu sesión iniciada en GitHub.   3) Un navegador (Chrome, Edge, etc.).

A Método rápido: subir solo lo que cambió

Usá este método casi siempre. Es el más ordenado: reemplazás solo los archivos que tocaste.

  1. Entrá a tu repositorio: github.com/gastonescudero83/tecnochips
  2. Entrá a la carpeta donde está el archivo que cambió. Ejemplo: si cambiaste css/asistente.css, hacé clic en la carpeta css. Si cambiaste algo del asistente, entrá a js y después a asistente.
  3. Arriba a la derecha, clic en Add file ▾ (en español: Agregar archivo ▾) y elegí Upload files (Subir archivos).
  4. Arrastrá desde tu PC el o los archivos actualizados al recuadro grande (o clic en choose your files / elige tus archivos).
    ⚠️ El archivo debe llamarse exactamente igual que el que ya está (mismas mayúsculas). Así lo reemplaza en vez de duplicarlo.
  5. Bajá hasta Commit changes (Confirmar cambios). En el primer casillero escribí una notita corta de qué cambiaste. Ejemplo: Actualizo el robot del asistente.
  6. Dejá marcada la opción Commit directly to the main branch (Confirmar directamente en la rama principal).
  7. Clic en el botón verde Commit changes (Confirmar cambios). ¡Listo, ya está subido! ✅
Consejo Si cambiaste varios archivos que están en la misma carpeta, podés arrastrarlos todos juntos y subirlos en un solo paso. Si están en carpetas distintas, hacé un envío por cada carpeta (repetí los pasos 2 a 7).

B Método completo: subir todo de nuevo

Usalo solo si cambiaste muchos archivos a la vez y preferís reemplazar todo.

  1. Entrá a tu repositorio (la página principal): github.com/gastonescudero83/tecnochips
  2. Clic en Add file ▾ → Upload files.
  3. En tu PC, abrí la carpeta tienda-online y entrá adentro. Seleccioná todo lo que está dentro (podés usar Ctrl + A) y arrastrá esa selección al recuadro.
  4. Escribí una nota (ej: Actualización general) y clic en Commit changes.
⚠️ Muy importante para no ensuciar el repositorio Arrastrá el contenido de la carpeta (los archivos y subcarpetas que están adentro), NO la carpeta tienda-online entera. Si arrastrás la carpeta completa, GitHub crea una carpeta tienda-online dentro del repositorio (un duplicado), y la tienda no se actualiza bien. La regla es: entrá a la carpeta y arrastrá lo de adentro.

C Ver los cambios en la tienda

  1. Esperá alrededor de 1 minuto a que GitHub publique.
  2. Abrí tu tienda y recargá con Ctrl + F5 (fuerza a bajar la versión nueva).
  3. Si la tenés instalada como app (PWA) en el celular, cerrala del todo y volvé a abrirla.

Si hiciste un cambio y no lo ves, casi siempre es la memoria (caché) del navegador: volvé a recargar con Ctrl + F5.

Tranquilidad GitHub guarda el historial de todo. Si algo sale mal, nunca se pierde: siempre se puede volver a una versión anterior. No tengas miedo de probar.
